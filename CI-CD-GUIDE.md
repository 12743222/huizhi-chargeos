# ============================================
# HUIZHI ChargeOS - CI/CD 自动化部署方案
# ============================================
#
# 方案概述：
#   GitHub Actions 自动构建 → 打包 → 上传服务器 → 自动部署
#
# 效果：
#   每次 push 代码到 GitHub，自动完成全部部署流程
#
# ============================================
# 一、准备工作（只需做一次）
# ============================================
#
# 1. 创建 GitHub 仓库（如果还没有）
#    在 https://github.com 新建一个仓库
#    例如：yourname/huizhi-chargeos
#
# 2. 推送到 GitHub
#    打开终端，在项目目录执行：
#
#    cd F:\慧知\HUIZHI-ChargeOS-cloud
#    
#    # 如果是新仓库：
#    git remote add origin https://github.com/你的用户名/你的仓库名.git
#    git branch -M main
#    git push -u origin main
#
#    # 如果已有远程仓库：
#    git remote set-url origin https://github.com/你的用户名/你的仓库名.git
#    git push -u origin main
#
# 3. 配置 GitHub Secrets（项目设置里）
#    在 GitHub 网页上进入仓库 → Settings → Secrets and variables → Actions
#    添加以下两个密钥：
#
#    | 密钥名            | 值                |
#    |-------------------|-------------------|
#    | SERVER_HOST       | 8.130.122.177     |
#    | SERVER_PASSWORD   | Aa123456          |
#
# ============================================
# 二、自动部署
# ============================================
#
# 配置完成后，每次你执行下面的操作，都会自动部署：
#
#    git add .
#    git commit -m "修改了xxx"
#    git push
#
# 然后去 GitHub 仓库页面 → Actions 选项卡
# 就可以看到自动构建和部署的进度。
#
# ============================================
# 三、手动触发部署
# ============================================
#
# 在 GitHub 网页上：
#   仓库 → Actions → Build & Deploy → Run workflow
#
# ============================================
# 四、如果不想用 CI/CD，继续手动部署
# ============================================
#
# 还是可以用原来的方式：
#   右键 F:\慧知\deploy.ps1 → 用 PowerShell 运行
#   选 1 上传 → 选 2 SSH 登录 → 执行部署脚本
#
